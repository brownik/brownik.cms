# townE 리뉴얼 프로젝트 스프린트 계획

## 프로젝트 개요
- **목표**: JSP/Spring MVC 기반 시스템을 Next.js(App Router) + Spring Boot 3.x REST API로 전환
- **기간**: Phase 1 (12-16주, 약 3-4개월)
- **DB**: 기존 MariaDB 유지 (스키마 변경 최소화)
- **참고 문서**: `new_analysis_docs/18_Migration_Roadmap.md`

## 기술 스택
- **Frontend**: Next.js 14+ (App Router), TypeScript, Tailwind CSS, TanStack Query, Zustand
- **Backend**: Spring Boot 3.x, Java 17, Spring Data JPA, Spring Security (JWT), Querydsl
- **DB**: MariaDB (기존 스키마 유지)

## 모듈별 우선순위 (점진적 마이그레이션 전략)
1. **회원 관리 모듈** (우선순위 1) - Sprint 2-3
2. **게시판 모듈** (우선순위 2) - Sprint 4-6
3. **컨텐츠 관리 모듈** (우선순위 3) - Sprint 7-9
4. **기타 모듈** (우선순위 4) - Sprint 10-12

## Sprint 2: 회원 관리 모듈 - 백엔드 개발 (Week 2)

#### JWT 인증 필터

  - 인증 실패 시 401 응답
  - SecurityConfig 설정

#### 토큰 갱신 API
- **US-011**: 사용자는 토큰을 갱신하여 지속적으로 인증 상태를 유지할 수 있다.
  - 토큰 갱신 API 엔드포인트 생성 (POST /api/v1/auth/refresh)
  - Refresh Token 검증 및 새 Access Token 발급

#### 회원 정보 조회/수정 API
- **US-012**: 사용자는 본인의 회원 정보를 조회할 수 있다.
  - 회원 정보 조회 API (GET /api/v1/members/me)
  - 인증 확인 (JWT)

- **US-013**: 사용자는 본인의 회원 정보를 수정할 수 있다.
  - 회원 정보 수정 API (PUT /api/v1/members/me)
  - 본인 권한 확인
  - 입력 데이터 검증
