# 로그인 문제 분석 및 해결 방안

## 현재 문제점

1. **기존 시스템 구조 미반영**
   - 기존: 관리자(`/admin`)와 일반 사용자(`/home`) 로그인 분리
   - 현재: 단일 로그인 시스템만 구현

2. **비밀번호 암호화 방식 불일치**
   - 기존: SHA-256 (Salt 없음)
   - 현재: BCrypt 사용
   - **문제**: 기존 DB의 비밀번호와 호환되지 않음

3. **SNS 로그인 미구현**
   - 기존: 카카오, 네이버 로그인 지원
   - 현재: 미구현

4. **전화번호 로그인 미구현**
   - 기존: 전화번호로 로그인 가능
   - 현재: 미구현

## 해결 방안

### 1단계: 로그인 문제 즉시 수정
- 기존 DB의 SHA-256 비밀번호와 호환되도록 수정
- 또는 마이그레이션 전략 수립

### 2단계: 관리자/일반 사용자 로그인 분리
- `/api/v1/auth/login` - 일반 사용자 로그인
- `/api/v1/admin/auth/login` - 관리자 로그인
- 각각 다른 권한 체크

### 3단계: SNS 로그인 구현 (추후)
- 카카오 로그인 API 연동
- 네이버 로그인 API 연동

### 4단계: 전화번호 로그인 구현 (추후)
- 전화번호 인증 시스템 연동
